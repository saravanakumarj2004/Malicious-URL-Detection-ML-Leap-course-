How DefendURL Works

A full technical walkthrough of the deep learning architecture, data pipeline, and the web infrastructure powering the URL threat detection system.

üîÑ Detection Pipeline

1

URL Input & Validation

A user submits a URL via the web interface. The frontend JavaScript sends the URL as a JSON POST request to the /predict API endpoint running on the FastAPI backend server.

2

Character-Level Tokenization

The raw URL string is tokenized at the character level using a pre-trained Keras Tokenizer (saved as tokenizer.pkl). Every single character (letters, digits, slashes, dots, etc.) is mapped to a unique integer index from the vocabulary it learned during training.

3

Sequence Padding

The tokenized sequence is padded (or truncated) to a fixed length of 200 characters using pad_sequences. This ensures all inputs to the neural network are the same shape, regardless of URL length.

4

CNN Forward Pass (Inference)

The padded sequence is passed through the urlnet_model.h5 ‚Äî a Convolutional Neural Network. The model detects suspicious character n-gram patterns (like login.php, .xyz, IP-only domains, etc.) and computes a raw threat probability score between 0 and 1.

5

Decision & Response

The model output is interpreted: a score > 0.5 is classified as Defacement / Malicious. A score ‚â§ 0.5 is classified as Benign / Safe. The result, score, and label are returned as JSON to the frontend where it renders the animated result card.

üìä Training Details

üìÅ Dataset

The model was trained on benign_vs_defacement_urls.csv, a real-world dataset containing thousands of labeled URLs ‚Äî classified as either benign (safe) or defacement (malicious website alteration). The dataset was balanced using equal sampling of each class to prevent bias.

‚öôÔ∏è Training Config

Optimizer  : Adam
Loss       : Binary Crossentropy
Epochs     : 5
Batch Size : 64
Val Split  : 10%
Test Split : 20%
Max URL Len: 200 chars

üî§ Tokenization

Tokenization happens at the character level, not word level. This means the model analyzes every single character in the URL. This makes it robust to obfuscation tricks like l0gin or unusual TLDs like .xyz or .tk.
